🧵 织女的一天 - 功能说明文档
这是一款基于 LocalStorage 的无后端手工艺品/织物作品管理工具，适用于追踪针织、毛线编织等手工项目的进度、成本与参数。

1. 核心定位
一款 纯前端 的个人作品管理系统，特点是：

数据本地化：所有信息（图片、数据、进度）均保存在浏览器的 localStorage 中，无网络同步，隐私性强。
可视化管理：通过卡片网格展示作品缩略图、状态与进度条。
专业工具：内置小样密度计算工具，帮助用户根据小样数据推算目标尺寸所需针数/行数。
2. 页面结构与主要模块
A. 作品列表页（默认页）
统计卡片

作品数量：当前库内作品总数。
总成本：所有作品线材成本之和。
进行中 / 已完成：统计不同状态的作品数量。
顶部控制栏

搜索：实时搜索作品名称。
状态筛选：按“未开始/进行中/已完成”过滤。
标签筛选：动态生成所有作品标签，支持多标签过滤。
添加按钮：点击弹出“添加作品”表单。
作品网格展示

采用 CSS Grid 布局，响应式支持 2/4/6 列显示。
卡片内容：图片（若无显示占位文字）、名称、状态、进度条、投入时间。
操作按钮：卡片右下角有“编辑”和“删除”按钮。
点击卡片：进入详情页（默认只读模式）。
B. 作品详情/编辑页
表单结构

图片上传：支持本地图片预览，提供“删除图片”功能。
基础信息：名称、标签、投入时间、线材名称/成本/消耗。
状态信息：
状态（下拉框：未开始/进行中/已完成）。
进度条（百分比），联动逻辑：修改状态自动调整进度值，反之亦然。
创建/开始/完成时间。
其他信息：针具型号、横向/纵向密度、备注。
模式切换

查看模式：所有输入框禁用，隐藏保存按钮，防止误编辑。
编辑/添加模式：所有输入框可编辑，显示保存按钮。
数据处理

保存时自动生成/更新 id（基于时间戳）。
标签输入支持英文逗号分隔，自动转化为数组存储。
完成编辑后返回列表页并刷新展示。
C. 小样密度计算工具页
核心功能：根据用户提供的小样尺寸（宽×高）和针数/行数，计算每10cm的针密度，并推算目标尺寸所需的针数与行数。
使用流程：
填写小样数据（宽度、高度、针数、行数）。
（可选）填写目标尺寸（宽度、高度）。
点击“计算”，弹出结果卡片。
结果展示：
横向密度（针/10cm）。
纵向密度（行/10cm）。
所需针数与行数（若填写目标尺寸）。
3. 数据存储与技术细节
LocalStorage：所有作品数据（含 Base64 编码的图片）均存储在浏览器本地，key: 'works'。
图片处理：使用 FileReader 将上传图片转为 DataURL，直接嵌入 JSON 中。
响应式布局：使用 CSS Media Queries 动态调整网格列数，兼容手机端（2列）与电脑端（6列）。
无后端：页面内所有逻辑均通过原生 JS 实现，适合部署在 GitHub Pages 或本地文件直接打开。
4. 使用建议
数据备份：定期手动复制浏览器控制台中的 localStorage.getItem('works') 数据，或使用浏览器的导出功能防止数据丢失。
图片优化：由于图片以 Base64 形式存储，占用空间较大，建议上传时使用压缩过的图片。
小样记录：使用工具页的计算结果作为记录，建议复制结果保存到对应作品的“备注”中，便于后期查询。
